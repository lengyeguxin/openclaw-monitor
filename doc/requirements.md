## 1. 引言

### 1.1 项目背景

OpenClaw 是一个多智能体系统，拥有 180+ 个专业智能体分布在 `/home/my/.openclaw/agency-agents/` 目录下。随着智能体数量的增加，需要一个统一的监控台来：
- 实时掌握所有智能体的运行状态
- 跟踪项目从创建到完成的完整生命周期
- 管理任务分配和执行进度

### 1.2 项目目标

构建一个**多智能体监控台**系统，实现对 180+ 智能体的统一监控、项目进度跟踪和任务分配管理。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 智能体 (Agent) | OpenClaw 系统中的专业智能体，如需求分析师、后端架构师等 |
| 项目 (Project) | 一个完整的开发任务，包含多个阶段和任务 |
| 阶段 (Stage) | 项目的子阶段，如需求分析、系统设计、开发实现等 |
| 任务 (Task) | 阶段内的具体工作项，可分配给智能体执行 |
| 监控台 (Monitor) | 本系统，用于监控智能体、项目和任务 |

## 2. 功能需求

### 2.1 智能体状态监控 (FR-001)

#### 2.1.1 功能描述

系统需要展示所有智能体的实时状态，支持 180+ 智能体的统一管理。

#### 2.1.2 详细需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| FR-001-1 | 展示智能体列表，包含名称、描述、分类、当前状态 | P0 |
| FR-001-2 | 支持按状态筛选（空闲/任务中/错误/离线） | P0 |
| FR-001-3 | 支持按分类筛选（产品、工程、设计、测试等） | P1 |
| FR-001-4 | 支持关键词搜索（名称、描述） | P1 |
| FR-001-5 | 显示智能体当前执行的任务信息 | P0 |
| FR-001-6 | 支持从 `/home/my/.openclaw/agency-agents/` 批量导入智能体 | P0 |
| FR-001-7 | 智能体状态实时更新（通过API上报） | P0 |
| FR-001-8 | 智能体历史任务记录查询 | P2 |

#### 2.1.3 状态定义

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `idle` (空闲) | 智能体未执行任务 | 默认状态，任务完成后 |
| `running` (任务中) | 智能体正在执行任务 | 接收到任务分配时 |
| `error` (错误) | 智能体执行出错 | 任务执行异常时 |
| `offline` (离线) | 智能体不可用 | 长时间未上报状态 |

---

### 2.2 项目进度监控 (FR-002)

#### 2.2.1 功能描述

系统需要支持项目→阶段→任务的层级结构，实现项目全生命周期的进度跟踪。

#### 2.2.2 详细需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| FR-002-1 | 项目列表展示，包含名称、描述、当前状态、进度百分比 | P0 |
| FR-002-2 | 项目详情页，展示项目基本信息和阶段列表 | P0 |
| FR-002-3 | 阶段管理：支持创建、编辑、删除阶段 | P0 |
| FR-002-4 | 阶段状态流转：待开始 → 进行中 → 已完成 | P0 |
| FR-002-5 | 自动计算项目进度（基于阶段和任务完成情况） | P0 |
| FR-002-6 | 显示当前活跃阶段和待处理任务数 | P1 |
| FR-002-7 | 支持项目状态筛选（草稿/进行中/已完成/已归档） | P1 |

#### 2.2.3 项目状态定义

| 状态 | 说明 |
|------|------|
| `draft` (草稿) | 项目创建初期，尚未启动 |
| `active` (进行中) | 项目已启动，正在执行中 |
| `completed` (已完成) | 所有阶段和任务已完成 |
| `archived` (已归档) | 项目已结束并归档 |

#### 2.2.4 阶段状态定义

| 状态 | 说明 |
|------|------|
| `pending` (待开始) | 阶段尚未开始 |
| `active` (进行中) | 阶段正在执行 |
| `completed` (已完成) | 阶段内所有任务完成 |
| `blocked` (阻塞) | 阶段因依赖未满足而阻塞 |

---

### 2.3 任务分配跟踪 (FR-003)

#### 2.3.1 功能描述

系统需要支持任务的创建、分配、执行跟踪，实现任务与智能体的关联管理。

#### 2.3.2 详细需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| FR-003-1 | 任务列表展示，包含名称、所属项目/阶段、优先级、状态 | P0 |
| FR-003-2 | 任务详情页，展示完整任务信息和执行智能体 | P0 |
| FR-003-3 | 支持创建任务，指定所属项目和阶段 | P0 |
| FR-003-4 | 支持分配智能体到任务（单/多智能体） | P0 |
| FR-003-5 | 显示任务执行智能体的实时状态 | P0 |
| FR-003-6 | 支持任务优先级设置（低/中/高/紧急） | P1 |
| FR-003-7 | 支持任务依赖关系设置 | P2 |
| FR-003-8 | 任务进度更新和完成标记 | P0 |
| FR-003-9 | 任务执行历史记录 | P2 |

#### 2.3.3 任务状态定义

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| `pending` (待处理) | 任务已创建，等待执行 | 任务创建后 |
| `running` (执行中) | 任务正在执行 | 智能体开始执行时 |
| `completed` (已完成) | 任务成功完成 | 智能体完成任务时 |
| `failed` (失败) | 任务执行失败 | 执行异常或超时 |
| `cancelled` (已取消) | 任务被取消 | 手动取消 |

#### 2.3.4 优先级定义

| 优先级 | 说明 | 颜色标识 |
|--------|------|----------|
| `low` (低) | 可延后处理 | 灰色 |
| `medium` (中) | 正常优先级 | 蓝色 |
| `high` (高) | 优先处理 | 橙色 |
| `urgent` (紧急) | 立即处理 | 红色 |

## 6. 数据模型

### 6.1 实体关系图

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   Agent     │       │   Project   │       │    Stage    │
│   (智能体)   │◄──────│   (项目)    │◄──────│   (阶段)    │
└──────┬──────┘       └─────────────┘       └──────┬──────┘
       │                                            │
       │                                            │
       │       ┌─────────────┐                    │
       └──────►│    Task     │◄───────────────────┘
               │   (任务)    │
               └─────────────┘
```

### 6.2 智能体 (Agent)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 智能体唯一标识，如 `agent_001` |
| name | string | 是 | 智能体名称，如 "需求分析师" |
| description | string | 否 | 智能体描述 |
| category | string | 否 | 分类，如 product/engineering/design |
| status | enum | 是 | 状态：idle/running/error/offline |
| capabilities | array | 否 | 能力标签列表 |
| config | json | 否 | 配置信息（模型、参数等） |
| current_task_id | string | 否 | 当前执行任务ID |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

**示例**：
```json
{
  "id": "agent_001",
  "name": "需求分析师",
  "description": "负责需求分析和文档编写",
  "category": "product",
  "status": "idle",
  "capabilities": ["analysis", "documentation"],
  "config": {
    "model": "gpt-4",
    "temperature": 0.7
  },
  "current_task_id": null,
  "created_at": "2026-03-23T10:00:00Z",
  "updated_at": "2026-03-23T10:00:00Z"
}
```

| name | string | 是 | 任务名称 |

| status | enum | 是 | 状态：pending/running/completed/failed/cancelled |
| priority | enum | 是 | 优先级：low/medium/high/urgent |
| progress | number | 是 | 进度百分比，0-100 |
| estimated_hours | number | 否 | 预估工时 |
| actual_hours | number | 否 | 实际工时 |
| assigned_agent_ids | array | 否 | 分配的智能体ID列表 |
| dependencies | array | 否 | 依赖任务ID列表 |
| started_at | datetime | 否 | 开始时间 |
| completed_at | datetime | 否 | 完成时间 |
| metadata | json | 否 | 扩展元数据 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

### 6.7 实体关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| Project → Stage | 1:N | 一个项目包含多个阶段 |
| Stage → Task | 1:N | 一个阶段包含多个任务 |
| Task → Agent | N:M | 一个任务可分配给多个智能体，一个智能体可执行多个任务 |
| Agent → Task | 1:1 (当前) | 一个智能体当前只能执行一个任务 |
