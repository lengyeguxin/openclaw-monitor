## 📁 项目根目录

```

├── 项目A/                    # 项目目录
│   ├── doc/                  # 文档
│   ├── code/                 # 代码
│   ├── data/                 # 数据
│   ├── output/               # 输出产物
│   └── tmp/                  # 临时文件 (任务完成后清理)

│   └── ...
```

3. 不同项目的目录结构可以不同（灵活调整）

## 🔄 子智能体调用规范 ⚠️ 重要

**工作流程**：

1. **接收任务** → 老大给我分配任务
2. **创建目录** → 在 `ai-project/` 下创建项目目录
3. **交给 orchestrator** → 调用 agents-orchestrator
4. **跟踪进度** → 监控任务执行情况
5. **汇报结果** → 向老大汇报完成情况

**调用方式**：

```bash
# 1. 先创建项目目录
mkdir -p /home/my/.openclaw/ai-project/项目名称/{doc,code,data,tmp}

# 2. 再调用 orchestrator
sessions_spawn(
  agentId="agents-orchestrator",
  cwd="/home/my/.openclaw/ai-project/项目名称/",
  task="老大交办的任务描述..."
)
```

**我的职责**：接收任务、创建目录、交给 orchestrator、跟踪进度、汇报结果

**orchestrator 职责**：任务分解、选择子智能体、协调协作、整合结果

**绝对禁止**：
- ❌ 不创建项目目录就调用 orchestrator
- ❌ 不指定 cwd
- ❌ 直接调用其他子智能体
